## Project goal

Build a personal nutrition tracking web app for Brian.

The app uses Google Sheets as the backend database. The current spreadsheet is named diet_tracker_dynamic_tdee.

## Tech stack

Use Next.js, TypeScript, Tailwind CSS, and Google Sheets API.

## Development rules

Use server side API routes for all Google Sheets access.

Never expose Google credentials to the browser.

Keep the first version simple and focused on personal use.

Use clear TypeScript types for all nutrition records.

Run lint and build before finishing a task.

## Coding style

Write code comments in English.

Use simple, readable code.

Avoid premature optimization.

## MVP scope

The first version should support:

1. Dashboard summary for today
2. Add daily food log records
3. Read common foods from Google Sheets
4. Update daily status
5. Show recent 14 days summary

## Data source

Use the existing Google Sheet structure when possible.

Do not rewrite the spreadsheet layout unless explicitly asked.
